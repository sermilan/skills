# 政策文件拆解 Skill

## Overview

本 Skill 将中国网络安全与数据保护领域的政策文件（法律、行政法规、部门规章、国标/行标）按照 **COU (Compliance Obligation Unit)** 方法论进行结构化拆解，输出三层产物：COU-R 原始义务单元、场景聚合文件、可视化 HTML 报告。

核心思想：**法条 → 五要素义务原子 → 场景聚合 → 合规视图**。

## Dependencies

None.（纯 Agentic 工作流，无需外部工具依赖。）

## Quick Start

**输入**：一份中国网络安全/数据保护政策文件的 Markdown 全文（含条款编号）。

**Agent 执行流程**：
1. 通读全文，判定文档层级（法律/行政法规/部门规章/国标/行标），确定 base_weight
2. 逐条拆解为 COU-R 单元（主体/动作/客体/条件/权重）
3. 按合规工作流或数据生命周期聚合为场景文件
4. 生成可视化 HTML 合规拆解报告

## Workflow

### Phase 1：文档层级判定与权重基准

在提取 COU 前，先判定文档的法律层级，确定 `base_weight`：

| 层级 | 类型 | base_weight | 示例 |
|------|------|-------------|------|
| 法律 | 全国人大及其常委会制定 | 10 | 《数据安全法》《个人信息保护法》 |
| 行政法规 | 国务院令 | 10 | 《关键信息基础设施安全保护条例》 |
| 部门规章 | 部委令/联合令 | 8 | 《网络数据安全风险评估办法》 |
| 国标 GB/T | 推荐性国家标准 | 6.4–10.4（按等级） | GB/T 22239-2019 |
| 行标 GA/T | 公安行业标准 | 6.4–10.4（按等级） | GA/T 2380-2026 |

### Phase 2：COU-R 逐条提取

#### 2.1 COU 五要素

每条义务条款解构为：

| 要素 | 说明 | 提取规则 |
|------|------|---------|
| **主体 (Subject)** | 谁承担义务 | 从条款中提取具体主体（重要数据处理者/网络运营者/评估机构/有关主管部门等），禁泛用"数据处理者" |
| **动作 (Action)** | 做什么/不得做什么 | 优先使用原条款的模态词：应当/不得/可以/有权/鼓励 + 具体行为 |
| **客体 (Object)** | 动作指向什么 | 条款要求的具体事项，≤120字符 |
| **条件 (Condition)** | 何时触发 | 如"定级为第3级时""年度风险评估完成后20个工作日内""无特定触发条件" |
| **权重 (Weight)** | 合规重要度 | `base_weight × factor`，factor 由模态词决定 |

#### 2.2 权重因子 (weight_factor)

| 模态词 | factor | 说明 |
|--------|--------|------|
| 应当/应 | 1.5 | 强制性义务，合规权重最高 |
| 不得 | 0.5 | 禁止性义务 |
| 可以/有权 | 1.0 | 授权性条款 |
| 鼓励 | 0.7 | 倡导性条款 |
| 按照/参照/定义/信息 | 0.8–1.0 | 宣示性/定义性条款 |

#### 2.3 文件命名规范

```
COU-R-{来源前缀}-{条款}-{动作标记}.md
```

- **标准类**：`COU-R-{标准号}-L{等级}-{章节}-{条款}.md`，如 `COU-R-2380-L3-6-5-1-a.md`
- **法律类**：`COU-R-{法典缩写}-{条款}-{动作}.md`，如 `COU-R-DSL-三十一-应.md`
- **部门规章类**：`COU-R-{规章缩写}-{条款}-{动作}.md`，如 `COU-R-NSRA-十六-应当-报送.md`

来源前缀取文档英文关键词首字母缩写，如 NSRA = Network Security Risk Assessment。

#### 2.4 COU-R 文件模板

```markdown
---

#### 2.5 数据域 (domains) 分类

根据条款涉及的安全领域标注，每条约 1–3 个 domain：

- 法律/规章类：`总则` `基本规定` `风险评估` `第三方评估` `报告管理` `监管合规` `监督检查` `执法合规` `处罚` `法律责任` `数据安全` `附则` `特殊规定`
- 技术标准类：`物理安全` `网络安全` `边界安全` `计算环境安全` `数据存储安全` `访问控制` `安全审计` `人员管理` `制度管理`

### Phase 3：场景聚合分析

COU 提取完成后，按合规工作流或数据生命周期将 COU-R 聚合为场景文件。

#### 3.1 场景分类方法

- **法律/规章类**：按合规工作流组织，如"年度评估执行""报告编制与报送""强制评估应对""监管检查""违规处置"
- **技术标准类**：按数据生命周期组织，如"数据收集""数据存储""数据传输""数据共享""数据销毁"

#### 3.2 场景文件模板

```markdown
---
title: "场景-{场景名称}"
type: "场景"
data_phase: "{阶段名}"
source: "{文档名} ({缩写} {年份})"
cou_count: {关联COU数}
---

# 场景：{场景名称}

> **来源**: {文档全称}
> **涉及 COU 数量**: {N}
> **适用主体**: {主体列表}

## 场景概述

{一段话描述该场景的合规全貌}

## 关联 COU

### {子场景/分组标题} (第{X}条)

- [[{COU-ID-1}]] — {摘要} (w={weight})
- [[{COU-ID-2}]] — {摘要} (w={weight})

## 合规要点

1. {要点1}
2. {要点2}
```

#### 3.3 必备场景

每个政策文件至少产出：
- **合规全景** — 全部 COU 覆盖的总览，含多主体视角路径图和时限总表
- **总则与特殊规定** — 适用范围、定义、排除条款
- 按工作流/生命周期的子场景（3–6个）

### Phase 4：可视化 HTML 报告

生成一份自包含的内联 CSS HTML 报告，包含：

1. **封面头部**：文档名称、发布机关、通过/发布/施行日期、法律层级
2. **合规义务总览**：COU 总数、条款覆盖、义务类型分布条形图、权重对照表
3. **主体义务矩阵**：各类主体 × COU 数 × 应当/不得/可/鼓励 × 义务强度评级
4. **条款义务浓度分布**：水平柱状图（降序）+ 功能域分组表格
5. **监管架构**：纯 CSS 层级卡片图（禁 ASCII art）
6. **报告流转/流程机制**：纵向时间线
7. **核心主体合规清单**：具体义务 + 时限 + 条款溯源
8. **违规处置路径**：渐进式处置升级图
9. **关键时间线**

#### 设计原则

- 全部内联 CSS（`style="..."`），禁 `<style>` 标签和 class 选择器
- 纯 CSS 布局，**禁止使用 ASCII art 字符画**
- 柱状图使用 `<div>` + 固定 px 宽度 + `white-space: nowrap`
- 时间线使用 `position: absolute` + 左侧竖线 + 圆形节点
- 所有文本元素加 `white-space: nowrap` 防止折行
- 配色体系按文档类型自适应（法律→深蓝；规章→蓝紫；标准→青绿）

### 常见义务主体

| 主体 | 说明 |
|------|------|
| 重要数据处理者 | 处理重要数据的网络数据处理者 — 本办法最核心规制对象 |
| 一般数据处理者 | 处理一般数据的网络数据处理者 — 鼓励性适用 |
| 核心数据处理者 | 处理核心数据的 — 按国家有关规定执行 |
| 网络运营者 | 网络的拥有者、管理者和服务提供者 |
| 评估机构 | 第三方数据安全风险评估服务机构 |
| 有关主管部门 | 行业主管/监管部门 |
| 国家网信部门 | 国家互联网信息办公室 |

### 典型时限参数

| 义务 | 时限 |
|------|------|
| 年度评估 | 每年1次 |
| 重要数据处理者报报告 | 评估完成后20个工作日内 |
| 主管部门通报网信部门 | 收到报告后10个工作日内 |
| 报告保存 | 至少3年 |
| 整改报告 | 整改完成后15个工作日内 |
| 主管部门报检查计划 | 每年1月底前 |
| 评估机构轮换 | 同一机构连续≤3次 |

## 表达风格指南

1. **主体精确化**：拒绝泛用"数据处理者"，必须从条款中提取具体主体称谓
2. **客体≤120字符**：保留原文核心语义，超出截断加 `...`
3. **条件具体化**：条件带有实际触发信息（时限/事件/状态），非空泛"无特定触发条件"
4. **权重可追溯**：`base_weight × factor = final_weight` 计算过程显式标注
5. **原文引用完整**：每条 COU 的原文引用应包含完整的条款原文句子

## 诚实边界

本 Skill 适用于：
- ✅ 中国网络安全与数据保护领域的法律、行政法规、部门规章
- ✅ GB/T 国标、GA/T 行标、YD/T 行标
- ✅ 条款编号清晰、有明确义务主体的政策文件

本 Skill 不适用于：
- ❌ 非中文政策文件
- ❌ 条款结构模糊、无明确义务条款的纯宣示性文件
- ❌ 非网络/数据安全领域的法规（如金融、医疗专项法规需调整 domain 体系）
- ❌ 法院判决书、行政处罚决定书等司法/执法文书
